## 还需要人工完成的项目

### 1. 云模型真实 key 验收

原因：需要真实 AIBuff / OpenAI-compatible API Key，Codex 不能创建、读取或代填密钥。

步骤：

1. 启动桌面版或浏览器开发版。
2. 在右侧模型配置中启用 Backup。
3. 选择 AIBuff 或 Custom。
4. 填入：
   - Base URL：`https://api.aibuff.cc/v1` 或你的兼容端点
   - Model：实际可用模型名
   - API Key：人工输入
5. 点击 Test connection。
6. 在画布选中对象，发送一次修订请求。
7. 验证返回消息下方显示实际 slot / preset / model。
8. 人为制造 Primary 可 fallback 错误，例如把 Primary base URL 改成不可连接地址。
9. 再发送请求，确认 Backup 接管，并显示 fallback reason。
10. 把 Backup 或 Custom 临时改成明确不支持图片输入的纯文本模型，再选中画布对象发送带截图请求，确认 UI 显示可读错误。
11. 把 Primary API Key 临时改错，Backup 保持有效，发送请求，确认不会自动切 Backup，并提示认证/配置错误。

通过标准：

- Test connection 成功。
- 文字请求能返回。
- 带选区截图的请求能返回修订提示词。
- Primary 出现连接失败、超时、429 或 5xx 时 Backup 接管。
- 401、模型不存在、请求格式错误不会误切 Backup。
- 纯文本模型收到截图时，错误提示可读，不泄露完整原始请求、API Key 或截图数据。
- 普通聊天当前为非流式响应；长回复等待整段返回属于首版已知取舍，流式显示留作后续。

### 2. Windows / Linux GUI 实机验收

原因：CI 已完成 Windows / Linux 打包，但 GUI、本地模型连接和系统路径行为需要真实机器确认。

步骤：

1. 从 GitHub Actions `Desktop package` artifact 下载对应平台产物。
2. 安装或解压启动。
3. 启动本机 Ollama 与 ComfyUI。
4. 在右侧服务状态确认：
   - Ollama connected
   - ComfyUI connected
   - Canvas directory 指向用户数据目录
5. 走一遍 M1 闭环：
   - 在画布创建对象或导入图片
   - 添加标注
   - 发送修订请求
   - 点击 Generate revision
   - 新图落到原图旁
6. 关闭应用并重开，确认画布恢复。
7. 测试导出：
   - Export JSON
   - Export ZIP
   - Export PNG
8. 测试 Canvas History restore。
9. 测试右键菜单 LocalArt 分组。

通过标准：

- 应用能启动。
- 本地服务状态正确显示。
- M1 闭环可完成。
- 画布可恢复。
- 导出文件可打开。
- 历史版本可恢复。
